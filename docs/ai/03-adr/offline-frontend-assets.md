# Offline Frontend Assets

## Status

Active requirement for ERPSMART local/customer deployments.

## Problem

The Core layout loads the global Vue runtime before the Vite application bundle. The Vite setup uses `@concordcrm/vite-plugin-global-vue`, therefore compiled bundles expect a browser global named `Vue`.

The upstream implementation generated this script URL from `Modules\Core\Application::vueSrc()`:

```text
https://unpkg.com/vue@<version>/dist/vue.global.prod.js
```

When internet or CDN access is blocked, the request fails and the whole UI breaks with:

```text
Vue is not defined
CreateApplication is not defined
```

## ERPSMART Rule

Do not rely on CDN assets for the core runtime. The Vue global runtime must be served locally from `public/vendor/vue`.

## Implementation

`modules/Core/app/Application.php` returns local asset URLs:

```php
asset('vendor/vue/vue.global.prod.js')
asset('vendor/vue/vue.global.js')
```

The files are generated from the installed npm package by:

```bash
docker compose exec node node scripts/sync-local-vue-assets.mjs
```

Expected files:

```text
public/vendor/vue/vue.global.js
public/vendor/vue/vue.global.prod.js
```

## Builder/RAG Note

The AI Builder must classify this as a platform-level dependency, not a Warehouse-specific issue. Any future module generated by the builder inherits this runtime requirement.

## Validation

Browser Network must not request:

```text
unpkg.com/vue
```

The app must load Vue from:

```text
http://localhost:8080/vendor/vue/vue.global.prod.js
```
